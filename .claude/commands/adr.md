# Architectural Decision Record (ADR)

We need to make an **Architectural Decision**. Draft a decision proposal with the following sections:

## ADR Template

### 1. Context
Describe the problem we are solving. What is the current situation? What constraints exist?

### 2. Decision
The proposed solution. Be specific about:
- Which technology/approach/pattern to use
- Why this over alternatives
- What changes are required

### 3. Consequences

**Pros:**
- List benefits

**Cons:**
- List drawbacks

**Changes Required:**
- Code changes needed
- Configuration changes
- Documentation updates to CLAUDE.md

### 4. Status
- **Proposed** / **Accepted** / **Deprecated** / **Superseded**

---

## After Decision is Accepted

Update CLAUDE.md Section 5 (Decision Log) with:

```markdown
| ADR-XXX | [Decision Title] | Accepted | [Date] |
```

Then commit as: `docs(adr): Add ADR-XXX [Brief Summary]`
